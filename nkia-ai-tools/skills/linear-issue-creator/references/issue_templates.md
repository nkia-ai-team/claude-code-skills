# Linear 작업 템플릿별 상세 가이드

이 문서는 8가지 작업 템플릿별 상세 마크다운 템플릿과 가이드를 제공합니다.

---

## 1. 빌드/배포 (Task 타입)

**자동 라벨:** "build"

**마크다운 템플릿:**
```markdown
## 배경
[왜 이 빌드/배포가 필요한지]

## 작업 설명
[무엇을 빌드/배포하는지]

## 배포 체크리스트
- [ ] Dockerfile 작성
- [ ] Jenkinsfile 작성
- [ ] 로컬 환경에서 API 호출 테스트
- [ ] Jenkins 빌드 성공 확인
- [ ] Nexus 이미지 업로드 확인
- [ ] 개발 서버 이미지 배포 및 검증

## 참고사항
[선택: 이미지 태그, 환경 설정, 관련 이슈]
```

**수집 정보:**
- 배경 (왜 필요한지)
- 작업 설명 (무엇을 빌드/배포하는지)
- 추가 체크리스트 항목 (선택)
- 참고사항 (선택 - 이미지 태그, 환경 설정, 관련 이슈)

## 7. 리서치 (Research 타입)

**자동 라벨:** "research"

**마크다운 템플릿:**
```markdown
## 배경
[왜 이 조사가 필요한지]

## 조사 목적
[답해야 할 핵심 질문]

## 조사 범위/항목
- 조사 영역 1
- 조사 영역 2
- 조사 영역 3

## 예상 결과물
- [ ] 결과물 1 (예: 기술 보고서)
- [ ] 결과물 2 (예: POC 구현)
```

**수집 정보:**

## 8. 버그 수정 (Bug 타입)

**자동 라벨:** "bug"

**마크다운 템플릿:**
```markdown
## 배경
[언제 발견되었는지, 영향도]

## 버그 설명
[무엇이 문제인지]

## 재현 방법
1. 단계 1
2. 단계 2
3. 단계 3

**예상 동작:** [예상되는 동작]
**실제 동작:** [실제로 발생하는 동작]

## 완료 조건
- [ ] 버그가 더 이상 재현되지 않음
- [ ] 테스트 케이스 추가됨
- [ ] 검증 완료

## 참고사항
[선택: 환경 정보, 로그, 스크린샷]
```

**수집 정보:**
- 배경 (언제 발견되었는지, 영향도)
- 버그 설명 (무엇이 문제인지)
- 재현 방법 (단계별)
- 예상 동작
- 실제 동작
- 완료 조건 (한 줄씩)
- 참고사항 (선택 - 환경 정보, 로그, 스크린샷)

## 제목 개선 가이드라인

좋은 제목의 특징:
- **행동 지향적** - 동사 사용
- **구체적이고 간결함** - 모호한 용어 피하기
- **범위와 영향이 명확함**

### 예시:

**빌드/배포:**
- ❌ "빌드" → ✅ "RCA 에이전트 v2.0 개발 서버 배포"
- ❌ "배포" → ✅ "추천 시스템 API 프로덕션 배포"

**데이터 작업:**
- ❌ "데이터 작업" → ✅ "Bank 데이터셋 전처리 및 라벨링"
- ❌ "데이터 수집" → ✅ "Q4 사용자 행동 데이터 수집 및 검증"

**평가:**
- ❌ "평가" → ✅ "RCA 모델 정확도 평가 (Acc@3)"
- ❌ "테스트" → ✅ "추천 알고리즘 A/B 테스트 결과 분석"

**기능 개발:**
- ❌ "기능 추가" → ✅ "리포트에 CSV 내보내기 기능 추가"
- ❌ "새 기능" → ✅ "사용자 대시보드에 실시간 알림 기능 구현"

**기능 개선:**
- ❌ "더 빠르게 만들기" → ✅ "사용자 검색 데이터베이스 쿼리 성능 최적화"
- ❌ "개선" → ✅ "로그인 플로우 UX 개선"

**리팩토링:**
- ❌ "코드 정리" → ✅ "인증 모듈 아키텍처 리팩토링"
- ❌ "리팩토링" → ✅ "레거시 결제 시스템 모던 아키텍처로 전환"

**리서치:**
- ❌ "조사" → ✅ "GraphQL vs REST API 성능 비교 조사"
- ❌ "리서치" → ✅ "새로운 ML 프레임워크 도입 가능성 연구"

**버그 수정:**
- ❌ "로그인 문제" → ✅ "OAuth 사용자 로그인 실패 수정"
- ❌ "버그 수정" → ✅ "결제 금액 계산 오류 수정"

## 작업 템플릿 → 이슈 타입 자동 매핑

| 작업 템플릿 | Linear 이슈 타입 |
|------------|----------------|
| 빌드/배포 | Task |
| 데이터 작업 | Task |
| 평가 | Task |
| 새로운 기능 개발 | Feature |
| 기능 개선 | Feature |
| 리팩토링 | Feature |
| 리서치 | Research |
| 버그 수정 | Bug |

---

## 라벨 자동 적용 규칙

### 템플릿별 기본 라벨:
- **빌드/배포** → "build"
- **데이터 작업** → "task"
- **평가** → "task"
- **새로운 기능 개발** → "feature"
- **기능 개선** → "improvement"
- **리팩토링** → "improvement"
- **리서치** → "research"
- **버그 수정** → "bug"

### 내용 기반 추가 라벨:
내용 분석을 통해 다음과 같은 추가 라벨 적용 가능:
- "backend" - 백엔드 관련
- "frontend" - 프론트엔드 관련
- "API" - API 관련
- "database" - 데이터베이스 관련
- "infrastructure" - 인프라 관련
- "security" - 보안 관련
- "performance" - 성능 관련
- "documentation" - 문서 관련
