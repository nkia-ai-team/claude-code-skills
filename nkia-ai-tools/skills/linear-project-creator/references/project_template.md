## 필수 수집 정보

1. **팀 이름** - Linear 팀 선택 또는 입력
2. **프로젝트 이름** - 프로젝트의 이름
3. **프로젝트 요약** - 한 줄 요약 (최대 255자)
4. **프로젝트 설명** - 프로젝트에 대한 상세 설명
5. **프로젝트 목표** - 달성하려는 주요 목표들

## 선택 정보

6. **우선순위** - No priority(0), Urgent(1), High(2), Medium(3), Low(4) 중 선택
7. **프로젝트 리드** - 프로젝트 리더 이름, 이메일, "me", 또는 비워두기
8. **시작일** - 프로젝트 시작 예정일 (YYYY-MM-DD 형식)
9. **목표일** - 프로젝트 완료 목표일 (YYYY-MM-DD 형식)

## 우선순위 매핑

- No priority → 0
- Urgent → 1
- High → 2
- Medium(또는 Normal) → 3
- Low → 4

## 템플릿 섹션별 가이드

### 프로젝트 개요
프로젝트의 목적, 배경, 주요 가치를 설명합니다.

### 프로젝트 목표
구체적이고 측정 가능한 목표를 번호로 나열합니다.

### 주요 기능/범위
프로젝트를 Phase별로 나누어 체크리스트 형태로 작성합니다.
- Phase 1: 초기 MVP 또는 핵심 기능
- Phase 2: 추가 기능 및 개선
- Phase 3: 고급 기능 및 최적화

### 기술 스택
사용할 기술들을 카테고리별로 정리합니다.

### 팀 구성
프로젝트에 참여하는 팀원들의 역할을 명시합니다.

### 일정
시작일과 목표일이 제공된 경우, 기간을 계산하고 주요 마일스톤을 설정합니다.

### 성공 지표
프로젝트의 성공을 측정할 수 있는 KPI나 메트릭을 정의합니다.

### 위험 요소 및 대응 방안
예상되는 리스크와 대응 계획을 테이블 형태로 정리합니다.

### 참고 자료
관련 문서, 디자인 파일, 요구사항 문서 등의 링크를 포함합니다.
